// PROTOCOL FEATURES

Every layer,
purpose-built.

Six contracts, ~3000 lines of Solidity. Every piece designed to eliminate the weaknesses of existing on-chain options protocols.

∫ BlackScholes.sol

On-Chain Black-Scholes

Premium computed in pure Solidity using WAD (1e18) fixed-point arithmetic. Abramowitz & Stegun polynomial approximation for N(x) CDF, accurate to 1e-7. No external price oracles — all math is on-chain and deterministic.

uint256 d1 = (lnSK + vol2t/2) / volSqrtT; // WAD arithmetic
call = spot * N(d1) - strike * N(d2); // C = S·N(d1) - K·N(d2)
◈ VolatilityOracle.sol

4-Chain Vol Index

Reactive Network aggregates realized vol from 288 samples/day per chain. Ethereum (35%), Arbitrum (30%), Base (20%), BSC (15%). Staleness protection: reverts if older than 1 hour.

⇄ OptionsHook.sol

Uniswap V4 Hook

Trades execute through Uniswap V4 beforeSwap hook. hookData encodes strike, expiry, call/put, quantity. BeforeSwapDelta collects premium in the same transaction.

⏱ Reactive Cron

Automated Settlement

Reactive Network cron calls settleExpiredSeries() at expiry. ITM holders call claimSettlement() to receive USDC payout. Zero keepers. Zero infrastructure.

⬡ CollateralVault.sol

Collateral Vault

USDC vault with share-based accounting. Writers earn 17.3% APY from option premiums. Utilization tracking prevents over-leverage. Premiums increase NAV-per-share passively.

// CIRCUIT BREAKERS & SAFETY MECHANISMS

Adversarial design.
Every edge case is closed.

Four independent safety layers prevent stale-price exploits, oracle spoofing, vault insolvency, and unauthorized hook deployment.

⏱
Vol Staleness Hard-Revert

The moment VolatilityOracle.getVolatility() is called, it checks the timestamp. If the last push is older than 1 hour, the entire transaction reverts. A stale vol feed would let arb bots extract risk-free value from mispriced options — this makes that impossible.

require(
  block.timestamp - lastUpdated < 1 hours,
  "VOL_STALE"
); // VolatilityOracle.sol
⬡
Collateral Utilization Cap

Before every option sale, OptionsHook queries available vault liquidity. If the locked collateral ratio would be breached, the trade reverts. The vault is always solvent: every ITM payout is pre-funded in USDC at the moment of sale.

require(
  vault.availableLiquidity(USDC) >= notional,
  "INSUFFICIENT_COLLATERAL"
); // OptionsHook.sol
🔑
Authorized Oracle Role

Volatility writes are gated behind the reactiveRelayer address set at deploy time. Only the Reactive Network relayer can push vol data — no one else can manipulate the vol index, regardless of gas or MEV.

require(
  msg.sender == reactiveRelayer,
  "NOT_RELAYER"
); // VolatilityOracle.sol
⇄
Hook Permission Bits

The Uniswap V4 Pool Manager validates hook permissions by inspecting the hook's address. Voltaire uses bitmask 0x88: beforeSwap=true and beforeSwapReturnDelta=true. Any deployment to an address encoding extra permissions reverts at the pool level — no privilege escalation possible.

Hooks.Permissions({
  beforeSwap: true,
  beforeSwapReturnDelta: true,
  // all others: false → 0x88
})
